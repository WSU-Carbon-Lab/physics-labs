# Siglent SDG Series Programming Context via NI-VISA & Python

## 1. Communication Interface Setup

To communicate with the SDG instrument using Python, the `PyVISA` library is recommended, utilizing NI-VISA as the backend. The device communicates using SCPI (Standard Commands for Programmable Instruments) strings.

### VISA Resource Strings

* **USB (USBTMC):** `USB0::<VendorID>::<ProductID>::<SerialNumber>::INSTR`
*
*Example:* `USB0::0xF4EC::0x1101::#15::INSTR`.




* **LAN (VXI-11):** `TCPIP0::<IP_Address>::INSTR`
*
*Example:* `TCPIP0::10.11.13.32::INSTR`.




*
**LAN (Socket):** Socket API can be used directly with port **5025**.



### Python (PyVISA) Connection Skeleton

```python
import pyvisa as visa

rm = visa.ResourceManager()
# Open connection using the specific resource string
device = rm.open_resource('USB0::0xF4EC::0x1102::SDG7ABAQ5R0010::INSTR')

# Configure timeout (e.g., 5000ms)
device.timeout = 5000

# Write a command (e.g., turn output on)
device.write('C1:OUTP ON')

# Query the device ID
response = device.query('*IDN?')
print(response)

device.close()

```

Note: SCPI commands usually end with a new line character (`\n`). PyVISA `write/query` methods typically handle termination automatically.

## 2. Command Syntax & Notation

* **Headers:** Commands are case-insensitive. They have a long form (e.g., `Basic_WaVe`) and a short form (e.g., `BSWV`). Capital letters in documentation indicate the short form.


*
**Channel Selection:** Commands typically start with a channel prefix: `C1:` or `C2:` (e.g., `C1:BSWV`).


*
**Parameters:** Multiple parameters are separated by commas.


*
**Queries:** A question mark `?` following the header indicates a query (e.g., `C1:BSWV?`).

## 3. IEEE 488.2 Common Commands

| Command | Description | Return Format |
| --- | --- | --- |
| `*IDN?` | Identification Query. Returns manufacturer, model, serial number, and firmware version.

 | <br>`Siglent Technologies,<Model>,<Serial>,<Firmware>`.

 |
| `*RST` | Reset. Initiates a device reset and recalls default setup.

 | None |
| `*OPC` | Operation Complete. Sets the OPC bit (bit 0) in the Event Status Register (ESR) when previous commands complete.

 | <br>`1`.

 |

## 4. Signal Output Commands

Controls the physical output BNC connectors.

### Output Control (`OUTPut`)

*
**Command:** `<channel>:OUTPut <parameter>,<value>`.


*
**Query:** `<channel>:OUTPut?`.


* **Parameters:**
*
`ON` / `OFF`: Enable or disable the output.


* `LOAD`: Set load impedance. Values: `50` (50Ω), `HiZ` (High Z), or a specific value (e.g., `10000`).


* `PLRT`: Polarity. `NOR` (Normal) or `INVT` (Invert).




*
**Example:** `C1:OUTP ON,LOAD,50` (Turn Ch1 ON, set load to 50Ω).



---

## 5. Basic Waveform Commands (`BSWV`)

Sets parameters for basic waveforms like Sine, Square, Ramp, Pulse, etc..

*
**Command:** `<channel>:BaSic_WaVe <parameter>,<value>`.


*
**Query:** `<channel>:BaSic_WaVe?`.



Key Parameters

| Parameter | Description | Valid Values |
| --- | --- | --- |
| `WVTP` | Waveform Type | `SINE`, `SQUARE`, `RAMP`, `PULSE`, `NOISE`, `ARB`, `DC`, `PRBS`, `IQ` |
| `FRQ` | Frequency | Value in Hz (e.g., `1000`) |
| `PERI` | Period | Value in Seconds (e.g., `0.01`) |
| `AMP` | Amplitude (Vpp) | Value in Volts (e.g., `2`) |
| `OFST` | Offset | Value in Volts (e.g., `0`) |
| `PHSE` | Phase | `0` to `360` degrees |
| `DUTY` | Duty Cycle | `0` to `100` (%) - *Square/Pulse only* |
| `SYM` | Symmetry | `0` to `100` (%) - *Ramp only* |
| `WIDTH` | Pulse Width | Value in Seconds - *Pulse only* |
| `RISE` / `FALL` | Rise/Fall Time | Value in Seconds - *Pulse only* |
| `STDEV` | Standard Deviation | Value in Volts - *Noise only* |

*
**Example:** `C1:BSWV WVTP,SINE,FRQ,2000,AMP,3` (Set Ch1 to Sine, 2kHz, 3Vpp) .

## 6. Modulation, Sweep, and Burst

### Modulation (`MDWV`)

Sets modulation parameters (AM, FM, PM, etc.).

*
**Enable:** `C1:MDWV STATE,ON`.


*
**Type Selection:** `AM`, `FM`, `PM`, `PWM`, `ASK`, `FSK`, `PSK`, `DSBAM`.


* **Parameters:**
*
`SRC`: Source (`INT`, `EXT`).


*
`MDSP`: Modulating Wave Shape (`SINE`, `SQUARE`, etc.).


*
`FRQ`: Modulating Frequency.


*
`DEPTH`: AM Depth (0-120%).


*
`DEVI`: FM/PM/PWM Deviation.




*
**Example:** `C1:MDWV AM,MDSP,SINE,FRQ,100,DEPTH,50`.



### Sweep (`SWWV` / `SWEep`)

*
**Standard Models:** Use `<channel>:SweepWaVe` (`SWWV`).


*
**Enable:** `C1:SWWV STATE,ON`.


*
**Parameters:** `TIME` (Sweep time), `START` (Start Freq), `STOP` (Stop Freq), `SWMD` (LINE/LOG), `DIR` (UP/DOWN), `TRSR` (Trigger Source: EXT, INT, MAN) .




*
**SDG7000A Model:** Uses specific subsystem commands like `<channel>:SWEep <switch>` and `<channel>:SWEep:TYPE <type>`.



### Burst (`BTWV`)

Sets burst parameters.

*
**Command:** `<channel>:BursTWaVe <parameter>,<value>`.


*
**Enable:** `C1:BTWV STATE,ON`.


* **Parameters:**
*
`GATE_NCYC`: Mode (`GATE`, `NCYC` for N-Cycle).


*
`TIME`: Cycle count (1 to 50000 or `INF`).


*
`TRSR`: Trigger Source (`EXT`, `INT`, `MAN`).


*
`PRD`: Burst Period.


*
`DLAY`: Burst Delay.

## 7. Arbitrary Waveforms

### Selecting Waveforms (`ARWV`)

*
**Command:** `<channel>:ArbWaVe INDEX,<index>` or `NAME,"<name>"`.


*
**List Waveforms:** `STL?` returns a list of stored waveforms with their indexes (M0-Mn) and names.


*
**Example:** `C1:ARWV INDEX,2` (Selects "StairUp").



### Transferring Waveform Data (`WVDT`)

To download custom arbitrary data.

*
**Command:** `<channel>:WVDT <parameter>,<value>`.


* **Parameters:**
*
`WVNM`: Waveform Name.


*
`FREQ`, `AMPL`, `OFST`, `PHASE`: Basic properties .


*
`WAVEDATA`: Binary data block.




* **Python Usage:** Requires constructing a binary block command. See programming example in documentation.

## 8. System & Utility Commands

*
**Synchronization:** `<channel>:SYNC <state>` (Enable Sync Output).


*
**Screen Saver:** `SCreen_SaVe <time>` (Time in minutes or OFF).


*
**Buzzer:** `BUZZer <state>` (`ON` or `OFF`).


*
**IP Address:** `SYST:COMM:LAN:IPAD "<ip_address>"`.


*
**Virtual Key:** `VirtualKEY VALUE,<key_index>,STATE,1` (Simulate front panel key press).

## 9. Model-Specific Notes

*
**SDG7000A:** Supports enhanced subsystems for `SWEep` , `IQ` signals , and `SEQuence` generation. It also supports specific file operations and digital channels.


*
**X Series:** Some parameters (e.g., `RISE`/`FALL` times) availability may vary by model.
